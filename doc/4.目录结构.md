## 目录结构

### 应用目录结构

```
project 应用部署目录
 ├─app应用目录
 │ ├─common 公共模块目录（必需，可以放公用工具类、配置文件等）
 | │ ├─config 公共配置文件目录
 │ ├─index 模块目录(可更改)
 │ │ ├─config 配置文件目录
 │ │ ├─controller 控制器目录
 │ │ ├─service (非必需，名称可自定义 可以把一些业务逻辑提取出来放到此)
 │ │ ├─model 模型目录 (对应数据表做增删改查的逻辑)
 │ │ └─ ... 可自定义更多类库目录
 ├─extend 扩展目录 （可以放自定义扩展或第三方扩展）
 ├─public WEB 部署目录（对外访问目录）
 │ ├─static 静态资源存放目录(css,js,image)
 │ ├─index.php HTTP 入口文件
 │ ├─rpc.php RPC 入口文件
 ├─runtime 应用的运行时目录（可写，可设置）
 ├─vendor 第三方类库目录（Composer 包括框架）
 ├─composer.json composer 定义文件
 ├─README.md README 文件
```

### 框架目录结构

```
mll-framework
 ├─Cache 缓存组件 (目前支持：Memcached, File)
 ├─Common 公共组件 (包含 组件读取(Dir), 缓存队列(MemcacheQueue), 规则(rule))
 ├─Config 配置文件加载组件
 ├─Controller 控制器组件
 ├─Core 核心组件 (包含 服务容器, 路由调用)
 ├─Curl Curl组件
 ├─Db 数据库组件 (目前支持：PDO, Mongo)
 ├─Exception 异常处理组件 (包含 自定义错误处理 日志回收)
 ├─Log 日志组件 (日志记录目前支持：File, Cache)
 ├─Request 请求组件 (协议处理目前支持：Rpc、Http)
 ├─Response 响应组件
 ├─Rpc Rpc组件
 ├─Server 服务组件 (目前支持：Rpc、Http)
 ├─Session Session组件
 ├─View 视图组件
 ├─Cache.php 缓存静态调用类
 ├─Session.php Session静态调用类
 ├─base.php 环境常量
 ├─main.php 引导文件
 ├─Mll.php 定义了autoloader, route, rootpath等框架所必需的数据和方法
```
